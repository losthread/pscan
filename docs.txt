# Default: top 1000 TCP ports
python scanner.py 192.168.1.1

Sends: TCP SYN packet to port
Receives SYN-ACK: Port is OPEN → Complete handshake (send ACK) → Close connection
Receives RST: Port is CLOSED
Timeout (no response): Port is FILTERED (firewall blocking)

  # All ports
  python scanner.py 192.168.1.1 -a

  # Specific range
  python scanner.py 192.168.1.1 -p 1-1024

  # Verbose output
  python scanner.py 192.168.1.1 -v

  # Only open ports
  python scanner.py 192.168.1.1 --open

Sends: TCP SYN packet to port
Receives SYN-ACK: Port is OPEN → Send RST (don't complete handshake)
Receives RST: Port is CLOSED
Timeout (no response): Port is FILTERED

  # SYN scan (requires sudo) 
  sudo python scanner.py 192.168.1.1 -sS

Sends: UDP packet to port
Receives UDP response: Port is OPEN
Receives ICMP unreachable: Port is CLOSED or FILTERED
Timeout (no response): Port is FILTERED

  # UDP scan
  sudo python scanner.py 192.168.1.1 -sU

Sends: TCP ACK packet to port (pretends connection exists)
Receives RST: Port is UNFILTERED (firewall not blocking)
Timeout (no response): Port is FILTERED (firewall blocking)

  # ACK scan
  sudo python scanner.py 192.168.1.1 -sA

After port scan: Try to grab service banner (sends service running on port and its version)
Sends: Generic probe packets or known service queries
Receives: Service name/version string (e.g., "Apache/2.4.41", "OpenSSH_7.4")
Purpose: Detect running service versions on open ports

  # Aggressive with version detection
  python scanner.py 192.168.1.1 -A

python scanner.py -h
